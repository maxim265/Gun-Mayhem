## 2 Варианты использования

<a name="func1"/>

### 2.1 Создать новую игру
**Описание:** Вариант использования "Создать новую игру" позволяет пользователю создать новый игровой мир

**Предусловия:**: Пользователь находится в главном меню.

**Основной поток:**

1. Пользователь запускает игровое приложение и видит главное меню.
2. Пользователь выбирает опцию "Создать новую игру" из меню.
3. Система предоставляет пользователю доступ к настройкам новой игры.
4. Пользователь выбирает уровень сложности.
5. Пользователь подтверждает выбранные настройки.
6. Система генерирует новую игру с учетом указанных параметров.
7. Пользователь видит начальное состояние новой игры на экране.
8. Игра начинается, и пользователь может начать игровой процесс.
9. Варинат использования завершается.

**Постусловие**: Пользователь успешно создал новую игру и находится внутри нее, готовый играть.

<a name="func2"/>

### 2.2 Продолжить игру
**Описание:** Вариант использования "Продолжить игру" позволяет пользователю возобновить свою предыдущую игру

**Предусловие:** Пользователь должен иметь сохраненные игровые данные.

**Основной поток:**

1. Пользователь запускает игровое приложение и видит главное меню.
2. Пользователь выбирает опцию "Продолжить игру" из меню.
3. Система отображает список доступных сохраненных игр пользователя.
4. Пользователь выбирает конкретную игру из списка, которую он хочет продолжить.
5. Система загружает сохраненное состояние выбранной игры.
6. Пользователь видит игровой интерфейс с текущим состоянием игры.
7. Игра возобновляется с того момента, где пользователь остановился в предыдущей сессии.
8. Варинант использования завершается

**Альтернативный поток А1:**
1. Загрузка последнего автосохранения 
2. Варинант использования завершается

**Альтернативный поток А2:**
1. Отсутствие сохраненных игр 
2. Система прелдагает перейти в варианту использования "Создать новую игру" 
2. Варинант использования завершается

**Альтернативный поток А3:**
1. Система сообщает, что файл сохранения подтвержет и что она не может загрузить новую игру
2. Варинант использования завершается

**Постусловие:** Пользователь успешно продолжил предыдущую игру и находится внутри нее с текущим состоянием.

<a name="func3"/>

### 2.3 Выйти из игры
**Описание:** Вариант использования "Выйти из игры" завершает свою текущую игровую сессию и выходит из игры.

**Предусловие:** Пользователь находится внутри игры и активно играет.

**Основной поток:**

1. Пользователь находится внутри игры, играя в нее.
2. Пользователь принимает решение завершить текущую игровую сессию и выйти из игры.
3. В игровом интерфейсе, часто в меню или опции, есть выбор "Выйти из игры".
4. Пользователь выбирает опцию "Выйти из игры".
5. Система сохраняет текущее состояние игры, чтобы пользователь мог возобновить игру позже.
6. Система завершает текущую игровую сессию и выходит из игры, переводя пользователя в главное меню или экран завершения игры.
7. Варинант использования завершается

**Альтернативный поток А4:**
1. Подтверждение выхода
2. Варинант использования завершается

**Альтернативный поток А5:**
1. Сохранение прогресса перед выходом 
2. Варинант использования завершается

**Альтернативный поток А6:**
1. Ошибка сохранения игры
2. Варинант использования завершается

**Постусловие:** Пользователь успешно вышел из текущей игры и находится в главном меню.

### 2.4 Открыть сундук и забрать содержимое инвентаря
**Описание:** Вариант использования "Открыть сундук и забрать содержимое инвентаря" дает возможность пользователю пополнять свой инвентарь.

**Предусловие:** Пользователь находится рядом с сундуком внутри игры и имеет доступ к своему инвентарю.

**Основной поток:**

1. Пользователь находится внутри игры и видит сундук, который он хочет открыть и забрать из него предметы.
2. Пользователь выбирает сундук и кликает по нему.
3. Система показывает интерфейс открытия сундука, который включает в себя изображение сундука и список предметов, находящихся внутри.
4. Пользователь просматривает содержимое сундука и выбирает предметы, которые он хочет забрать в свой инвентарь.
5. Система перемещает выбранные предметы в инвентарь пользователя.
6. Варинант использования завершается

**Альтернативный поток А7:**
1. Инвентарь пользователя переполнен
2. Варинант использования завершается

**Альтернативный поток А8:**
1. Пользователь не может забрать предмедмет из-за ограничений
2. Варинант использования завершается

**Постусловие:** Пользователь успешно открыл сундук, просмотрел его содержимое и забрал выбранные предметы в свой инвентарь.

### 2.5 Закрыть сундук
**Описание:** Вариант использования "Закрыть сундук" завершает взаимодействие пользователя с сундуком и закрывает его после того, как тот  посмотрел его сожержимое

**Предусловие:** Пользователь открыл сундук и просмотрел его содержимое.

**Основной поток:**

1. Пользователь находится внутри игры и видит открытый сундук с просмотренным содержимым.
2. Пользователь принимает решение закрыть сундук и завершить взаимодействие с ним.
3. Пользователь выбирает опцию "Закрыть сундук" или аналогичную опцию в интерфейсе.
4. Система закрывает сундук и скрывает его содержимое от пользователя.
5. Варинант использования завершается

**Постусловие:** Пользователь успешно закрыл сундук и завершил взаимодействие с ним.

### 2.6 Использовать зелье
**Описание:** Вариант использования "Использовать зелье" описывает, как пользователь может взаимодействовать с зельем в игре, чтобы получить его эффект.

**Предусловие:** Пользователь имеет в своем инвентаре зелье, которое он хочет использовать.

**Основной поток:**
1. Пользователь находится внутри игры и имеет доступ к своему инвентарю, где он видит зелье, которое он хочет использовать.
2. Пользователь выбирает зелье в своем инвентаре.
3. Система показывает опции, доступные для выбранного зелья, такие как "Использовать" или "Выбросить".
4. Пользователь выбирает опцию "Использовать".
5. Система применяет эффект зелья к игровому персонажу
6. Пользователь видит результат использования зелья
7. Вариант использования завершается

**Альтернативный поток А9:**
1. Зелье нельзя применить в текущей ситуации.
2. Варинант использования завершается.

**Альтернативный поток А10:**
1. Система просит подтвердить пользователя, что он хочет применить зелье.
2. Варинант использования завершается.

**Постусловие:** Пользователь успешно использовал зелье и получил его эффект.

### 2.7 Поднять предмет с земли
**Описание:** Вариант использования "Поднять предмет с земли" позволяет пользователю подобрать предмет, который лежит на земле.

**Основной поток:**

2. Игрок приближается к предмету, чтобы находиться в достаточной близости для поднятия.
3. Игрок кликает на предмет, чтобы поднять его.
4. Система поднимает предмет и перемещает его в инвентарь игрока.
5. Вариант использования завершается.

**Альтернативный поток А11:**
1. Инвентарь переполнен, предмет остался лежать на земле.
2. Варинант использования завершается.

**Постусловие:** Игрок успешно поднял предмет с земли и теперь он находится в его инвентаре.

### 2.12 Атаковать врага
**Описание:** Вариант использования "Атаковать врага" позволяет пользователю осуществить атаку на врага для его уничтожения.

**Предусловие:** Игрок находится в игре и находится в ситуации, где есть враг, которого можно атаковать.

**Основной поток:**
1. Игрок нашел врага, который находится в достаточной близости для атаки.
2. Игрок готовится.
3. Игрок выбирает врага в качестве цели для атаки. 
4. Пользователь выбирает опцию "Атака", чтобы перейти к атаке.
5. Игровой персонаж атаковал.
6. Вариант использования завершается.

**Альтернативный поток А16:**
1. Враг находится вне радиуса атаки.
2. Система предостовляет иформацию о том, что противник далеко.
3. Варинант использования завершается.

**Альтернативный поток А17:**
1. Противник уклонился от атаки.
2. Варинант использования завершается.

**Постусловие:** Игрок успешно атаковал противника.

**Альтернативный поток А18:**
1. Пользователю не удалось уклонится от атаки.
2. Игровой персонаж получил урон.
2. Варинант использования завершается.
